# NOVYN Development Rules

This is a personal tool.

The first user is the founder.

Do not optimize for scale.

Do not introduce:

- auth
- AI agents
- microservices
- vector databases
- Redis
- queues
- notifications

Build:

1. import
2. search
3. timeline
4. insights
5. revisit

Use:
- Next.js
- TypeScript
- Tailwind
- Supabase

Keep functions small.

Keep architecture simple.

Always ask:

Does this feature help users:
- find saved content
- revisit content
- understand interests

If not:
do not build it.

Generate one commit message for every implementation.
